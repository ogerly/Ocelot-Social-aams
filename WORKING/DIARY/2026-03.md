### 2026-03-27 — Deep Analysis Session

Created comprehensive whitepaper suite (WP-001 through WP-009) covering every project area: architecture, backend, cypress, deployment, database (Neo4j), styleguide/design system, webapp, API interfaces, ecosystem. Two workpapers: version currency assessment (270 dependencies analyzed — 2 CRITICAL deprecated: neo4j-graphql-js, neode) and Supabase evaluation (recommendation: migrate to Supabase/PostgreSQL — app doesn't use graph-specific features, effort comparable to Neo4j upgrade but gets auth+storage+realtime+managed hosting). Key insight: packages/ui is fully modern (Vite 7, Vue 3, Tailwind 4) while webapp is 2 generations behind. UI migration 89% complete. Upgrade order matters: database first, then Vue 3/Nuxt 3.

### 2026-03-27 — Supabase-Strategie verfeinert

Tradeoff-Analyse geschärft. Entscheidung: Supabase ja, aber inkrementell — kein Big Bang. Reihenfolge: (1) Auth+Storage zuerst (geringes Risiko, sofortiger operativer Gewinn), (2) PostgreSQL parallel aufbauen und validieren, (3) Resolver migrieren, (4) Neo4j erst abschalten wenn alles grün ist. Kritischer Einwand: ActivityPub/Fediverse ist explizites Projektziel und ein Graph-Protokoll — Apache AGE (Graph-Extension für PostgreSQL) als Hedge auf dem Radar. pg_graphql reicht NICHT für die bestehende Business-Logik — Apollo Server bleibt, nur der Datenbankzugriff wechselt. Datenmigration als eigenen Milestone planen.
